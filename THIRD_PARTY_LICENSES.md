## Caveman (`.claude/` and IDE rule files)

The following files are derived from
[JuliusBrussee/caveman](https://github.com/JuliusBrussee/caveman). They are
vendored so caveman activates in every Claude Code session that opens this
repo (local CLI, Claude Code on the web, and the `claude.yml` GitHub Action)
without requiring a per-machine plugin install.

- `.claude/hooks/caveman-activate.sh` — derived from upstream `hooks/caveman-activate.js` and `skills/caveman/SKILL.md`
- `.claude/commands/caveman.md` — derived from upstream `commands/caveman.toml` and `skills/caveman/SKILL.md`
- `.claude/commands/caveman-review.md` — derived from upstream `commands/caveman-review.toml` and `skills/caveman-review/SKILL.md`
- `.claude/commands/caveman-commit.md` — derived from upstream `commands/caveman-commit.toml` and `skills/caveman-commit/SKILL.md`
- `.cursor/rules/caveman.mdc`, `.windsurf/rules/caveman.md`, `.clinerules/caveman.md`, `.github/copilot-instructions.md`, `AGENTS.md` — generated by upstream `tools/caveman-init.js` from `rules/caveman-activate.md`

To resync against upstream, see <https://github.com/JuliusBrussee/caveman> and
re-derive the files above. Statusline badge, mode tracker, `caveman-stats`,
`caveman-shrink` MCP proxy, and `cavecrew` subagents from the upstream plugin
are intentionally not vendored.
