## Key Differences from Other Endpoints:
1. Uses enumValues filter for status: `status = open`
2. No default sorting (propertyOrderings is empty)
3. Longer date range (6 months ahead)
4. Exchange-specific fields: submittedAt, submittedByPerson, appliedBy, requiredRefereeLevel
5. Has "active" and "open" referee name fields for each position

## Exchange Status Values:
- open: Available for application
- applied: Someone has applied (waiting for assignment)
- closed: Exchange completed or cancelled

## Referee Level Codes:
- N1: National level 1 (highest)
- N2: National level 2
- N3: National level 3
- N4: National level 4 (lowest)

## Referee Level Gradation Values:
- 1: Highest gradation for the position
- 2: Second tier
- 3: Third tier
